Glossaire méthodologique
------------------------

.. xglossary::
    :sorted:

    CaptureDesBesoins
        Phase consistant à recueillir les besoins du client via des interviews, des réunions,
        l'élaborations de scénarios. Cette phase est suivie d'une phase d'analyse des besoins.

    IHM
    InterfaceHommeMachine
        Une IHM est un ensemble de dispositifs matériels et logiciels permettant à un utilisateur de communiquer avec un système informatique

    IHMA
    InterfaceHommeMachineAbstraite
        Une IHMA est une représentation graphique d'un projet par modélisation des différentes interactions possibles entre les espaces virtuels 
        qui seront présent dans un projet


    IHMC
    InterfaceHommeMachoineConcrete
        Une IHMC est une représentation graphique qui fait la réalisation de l'IHMA permettant d'appliquer les espaces virtuels de l'IHMA.

    MDT
    ModeleDeTaches
        Un MDT est de produire une liste des tâches, une description des tâches et leurs interactions.

    ModeleConceptuel
        Dans le domaine des bases de données, le ModeleConceptuel consiste en une représentation graphique et structurée des informations mémorisées

    ModeleLogique
        Dans le domaine des bases de données, le ModeleLogique consiste à décrire la structure de données utilisée sans faire référence à un langage de programmation

    ModelePhysique
        Dans le domaine des bases de données, le ModelePhysique consiste à implanter les tables d'une base de données 

    Contrainte
        Une contrainte est une résolution de problèmes d'optimisation

    ContrainteDIntegriteReferentielle
        `Contrainte` permettant d'exprimer ...

    UML
        Unified Modeling Language. UML est un ensemble de langage de
        modélisation standard. Ce projet est basé sur l'utilisation de
        diagrammes de classes pour définir le `ModeleConceptuel`
        de l'application. Le lagage `OCL` est

    OCL
    ObjectConstraintLanguage
        OCL est un des langages de modélisation appartenant à `UML`.
        Il permet de définir des `Contrainte` s ne pouvant pas être
        exprimées dans le cadre de diagrammes de classes.
